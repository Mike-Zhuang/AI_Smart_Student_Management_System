# 高中AI管理辅助系统

一个面向高中真实校园场景的一体化协同平台，覆盖家校沟通、学业成长、生涯选科与班级治理，并内置国产大模型能力。

适用场景：

- 学校日常管理与班级协同
- 班主任日常治理与家校联动
- 校内试点上线与持续迭代

## 2. 系统能力概览

### 2.1 核心业务模块

1. 家校沟通系统
2. 生涯发展与选科建议
3. 学业成长追踪系统
4. 班级治理中心

### 2.2 AI 助手中心

- 支持智谱模型切换
- 支持模板化业务场景（选科、成长、家校）
- 支持会话历史、上下文续聊、结构化输出渲染
- 支持流式输出与免费/收费模型标记
- 模板接口仅返回用户说明与变量信息，不向普通用户暴露系统提示词或内部格式规范

### 2.3 管理与数据治理能力

- 审计日志查询与导出
- 模块级数据导出
- 导入结果回滚与批量删除
- 整班级联删除与删除前二次确认
- 班级治理内容维护
- 家长账号自动发放与补充创建
- 班级空间只读共享与组织架构折叠查看

## 3. 3 分钟快速上手

- Node.js >= 20
- npm >= 10

```bash

# 或

```


## 4. 账号与角色

内部使用建议：

- 线上环境不开放公开注册。
- 新账号由管理员/班主任/教师在后台导入数据后统一发放。
- 首次拿到系统分配的初始密码后，建议登录后立即在“我的账号”中修改密码。
- “我的账号”内置账号发放台账，能清楚看到登录账号、最近发放时间、是否仍可再次下载密码，以及历史导入/重置批次。
- 学生导入时会自动生成主家长账号；管理员、班主任、教师也可以在“我的账号”中继续为同一学生追加更多家长账号。
- 管理员、班主任、教师还可进入“组织架构”查看全校班级、班主任、科任教师和学生花名册。
- 学生、家长、教师、班主任、管理员均可进入“班级空间”查看班级公共信息（只读）。

本地演示环境仍保留示例账号，便于开发联调：

- admin / admin123
- teacher_zhang / teacher123
- head_li / head123
- parent_wang / parent123
- student_001 / student123

角色权限说明：

- admin：全功能
- teacher：家校沟通、成长查看、选科支持、组织架构、AI 助手
- head_teacher：班级治理中心 + 教师能力 + 组织架构 + 请假审批
- parent：家校沟通、孩子成长查看、请假确认、班级空间只读查看
- student：个人成长、请假提交、选科完善、班级空间只读查看

## 6. 核心数据说明

### 6.1 内置演示数据

- 学生规模：支持千人级样本（当前部署数据为 1000+）
- 多次考试成绩与趋势
- 成长画像与预警
- 公开专业选科要求数据

### 6.2 真实数据导入

当前版本默认采用 Excel/CSV 直传导入，不需要手工转换 JSON。

页面操作流程：

1. 下载对应模板（学生/成绩/教师班级）
2. 在 Excel 或 WPS 中填写 XLSX 模板，或继续使用 CSV
3. 回到“数据导入”上传 XLSX / CSV 文件
4. 查看导入结果统计与逐行错误提示
5. 若系统生成了新账号，请立即下载一次性账号发放单
6. 若当时忘记下载，可到“我的账号 → 账号发放批次记录”重新下载仍未改密账号

模板文件：

- apps/backend/templates/students-template.csv
- apps/backend/templates/exam-results-template.csv
- apps/backend/templates/teachers-template.csv
- apps/backend/templates/students-template.xlsx
- apps/backend/templates/exam-results-template.xlsx
- apps/backend/templates/teachers-template.xlsx

导入接口：

- POST /api/data-import/students（`multipart/form-data`，字段名 `file`）
- POST /api/data-import/exam-results（`multipart/form-data`，字段名 `file`）
- POST /api/data-import/teachers（`multipart/form-data`，字段名 `file`）

说明：

- 导入结果会返回 `total/imported/updated/failed/errors` 统计。
- 管理员、班主任、教师均可执行数据导入与账号发放。
- 学生导入会自动同步学生账号，并自动生成主家长账号；教师导入会自动同步教师账号。
- 导入页面同时提供学生、成绩、教师关系的批量删除入口，并新增整班级联删除，便于回滚错误数据。
- 账号重置密码也会自动形成新的发放批次，避免一次性密码只弹出一次后丢失。
- 所有删除操作都需要先确认，避免误触。
- 整班级联删除会保留账号发放批次审计记录；若操作者账号被删除，系统会自动把历史批次挂到“系统审计保留”身份，避免外键失败。
- 当前线上同步脚本不会上传本地 `app.db`；同时生产环境默认禁用演示种子数据，不会因为重启或部署把本地 1000 条测试数据重新灌进线上库。

## 8. 常见问题（强烈建议先看）

### 8.1 登录时报错：无法连接服务器 / CORS

排查顺序：

1. 确认打开的是正确地址
2. 确认请求是发往 /api 而不是 http://localhost:4000
3. 刷新浏览器缓存后重试

### 8.2 控制台出现 runtime.connect 错误

这通常来自浏览器扩展，不是本项目后端报错。建议：

1. 用无痕模式重试
2. 暂时禁用扩展后重试

### 8.3 登录页应不应该默认填账号密码

不应该。当前版本已修复为首次进入空表单，并支持密码显示/隐藏切换。

### 8.4 数据导入提示“失败行”如何处理

处理顺序：

1. 按提示查看具体行号、字段和原因
2. 对照模板修正后重新上传同一 CSV
3. 若仍失败，先用 1-3 行小样本验证再批量导入

### 8.5 中文导入乱码怎么办

当前版本已兼容：

1. UTF-8 CSV
2. GBK / GB18030 CSV
3. XLSX

系统现在会对“上传文件 -> 解析单元格/列名 -> 入库前清洗 -> 历史数据库平滑修复 -> 查询接口输出”整条链路统一做编码修复与规范化，重点处理：

- `锟斤拷`
- `瀛﹂敓...`
- `�`
- 考试名称异常学年值
- 班级、姓名、学科等字段的混乱繁简/半角问题

并且会在成绩数据清理、班级空间、组织架构、家校消息、账号台账等接口出口继续做乱码兜底修复。

若老师不熟悉 CSV，优先使用系统提供的 XLSX 模板即可。

### 8.6 家长代提交请假为什么会直接进入班主任审批

当前流程是：

- 学生提交：仍然按“学生提交 -> 家长确认 -> 班主任审批 -> 返校销假”
- 家长代提交：系统会直接跳过“家长确认”节点，进入“班主任审批”

这样可以避免“家长自己提交后还要再确认一次”的重复操作。
### 8.7 选科建议流式中断或没有完成事件怎么办

当前版本已经对流式链路做了专门加固：

1. 后端会尽量保证成功时发出 `complete`、失败时发出 `error`
2. 结构化场景下，前端不再把原始 JSON 当正文展示，而是显示面向老师的自然语言流式内容
3. 前端会兼容多段 `data:`、不同换行符和“只有最终结果没有明显 delta”的返回
4. 即使模型没有明确完成事件，只要最终结果已齐全，系统也会按成功结果落库并刷新历史
5. 若上游输出被截断，会明确提示“输出被截断 / 结构化结果未完成”，而不是只显示 `Load failed`

如果仍失败，优先检查 API Key、模型可用性与网络连通性。

### 8.8 AI 思考链里为什么还会出现隐式乱码

当前版本已把“进入模型前的文本净化”补齐到选科、成长和家校等 AI 场景：

1. 姓名、班级、兴趣方向、生涯目标、考试名称、学科名、补充信息都会先做修复
2. 若修复后仍疑似乱码，不再把脏文本原样喂给模型
3. 默认会降级成“暂无有效兴趣信息”等安全占位，避免模型自行猜测
### 8.9 模型真实联调结果怎么看

本地真实联调已验证：

1. `glm-5.1`：流式正文、思考过程、完成事件正常
2. `glm-5-turbo`：流式正文、思考过程、完成事件正常
3. `glm-4.7-flash`：当前更容易遇到上游限流，若偶发失败优先视为模型配额/限流问题，而不是前端解析崩溃

因此，如果老师现场演示，建议优先使用配额更稳定的模型；若使用免费模型，请预留一次重试空间。
