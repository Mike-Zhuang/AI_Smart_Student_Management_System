# 8分钟答辩脚本与操作清单

本稿用于创AI案例现场演示，默认演示人使用 `admin` 账号登录。

## 1. 分秒级讲稿（8分钟）

### 0:00 - 0:40 项目开场

讲稿：

“本系统面向中国大陆高中管理场景，聚焦四个高频痛点：家校沟通效率低、选课建议不可解释、学情预警滞后、教研管理缺少闭环。我们采用前后端分离架构，支持角色权限分层和国产大模型智谱接入，目标是让班主任、教师和家长在同一平台协同。”

操作：

1. 打开总览页。
2. 展示左侧模块导航与当前登录角色。

### 0:40 - 2:10 家校沟通闭环

讲稿：

“家校模块不是只发消息，而是做到‘发送-回执-审批-留痕’。管理员或教师可按角色推送通知，家长端可回执，班主任可审核请假并形成审计记录，便于后续追责和复盘。”

操作：

1. 进入“家校沟通”。
2. 发送一条测试消息（接收角色选家长）。
3. 在消息列表点击“标记已读”，展示状态由未读变已读。
4. 在请假审批区执行“同意”或“驳回”。
5. 点击“导出消息”“导出请假记录”。

### 2:10 - 3:50 生涯选课（可解释）

讲稿：

“选课推荐我们强调可解释。系统不仅给出推荐组合，还输出五维评分、证据链和反事实分析，避免‘黑箱建议’，让老师和家长可以理解建议依据。”

操作：

1. 进入“生涯选课”，选择学生和模型。
2. 点击“生成选课建议”。
3. 展示推荐历史。
4. 展示“理由可解释面板”：
   - 维度分（science/social/logic/language/stability）
   - 证据链
   - 反事实说明
5. 点击“导出推荐记录”。

### 3:50 - 5:00 学业成长追踪

讲稿：

“成长模块将成绩趋势、成长画像和风险预警合并展示，班主任可快速识别重点学生并制定干预策略。”

操作：

1. 进入“学业成长”。
2. 切换学生并展示趋势图。
3. 展示风险预警列表。

### 5:00 - 6:20 班主任工作台 + 教研管理

讲稿：

“班主任工作台提供细颗粒决策视图：待办漏斗、家长回执率、风险学生清单、最近操作轨迹，帮助班主任每天先做最重要的事。教研模块则覆盖任务创建、成果归集和绩效统计。”

操作：

1. 进入“班主任工作台”，按班级筛选。
2. 展示待办漏斗、回执率、风险学生、审计轨迹。
3. 点击“导出评比汇总”。
4. 切换到“教研管理”，新建一个任务并展示统计变化。
5. 点击“导出任务记录”。

### 6:20 - 7:30 智谱模型与提示词模板

讲稿：

“AI实验室支持模型切换和预置提示词模板。我们按业务场景内置模板，避免自由提问导致输出不稳定，保障建议结构化、可追溯。”

操作：

1. 进入“智谱模型”。
2. 选择场景模板（如生涯选课）。
3. 填写变量 JSON，调用模型。
4. 展示模板说明、输出规范和模型返回结果。

### 7:30 - 8:00 结尾

讲稿：

“总结一下：我们完成了四大模块的一体化落地，并通过角色权限、审计导出和模板化 AI 输出保障可管可控。该系统可用于评比演示，也可在后续接入真实校内数据持续迭代。”

## 2. 现场操作清单（Checklist）

1. 启动服务前检查：`npm run dev` 或 `pnpm run dev:pnpm`。
2. 演示账号预检：确认管理员账号可登录，且页面不出现公开注册入口。
3. API Key 预检：提前配置可用智谱 Key。
4. 关键页面预热：总览、家校、生涯、成长、班主任、教研、AI。
5. 导出按钮演示：至少演示一次下载成功。
6. 风险兜底：若外网不稳定，优先演示已落库历史记录与导出能力。

## 3. 评比证据导出清单

1. 审计日志：`/api/admin/audit-logs` 或 `/api/admin/export/audit-logs`。
2. 模块数据：`/api/admin/export/module/:module`。
3. 评比汇总：`/api/admin/export/evidence-report`。
4. AI标识说明：参考 `docs/ai-generated-content-policy.md`。
5. 对照核查：参考 `docs/evaluation-checklist.md`。

## 4. 应急讲法（30秒）

若模型接口临时不可用，可使用以下话术：

“AI调用依赖外部模型服务，当前网络波动导致实时调用受限。系统本身已具备模板化提示词、调用审计和历史结果展示能力，核心管理流程与可追溯机制不受影响。”
